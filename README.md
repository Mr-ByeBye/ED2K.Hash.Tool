# ED2K Hash Tool

`ED2K Hash Tool` 是一个面向 Windows 的桌面工具，用来批量计算文件的 ED2K 链接，并对计算结果、历史记录和资源库进行统一管理。

## 项目简介

这个项目最早有旧版 WinForms 实现，后来重构为 WPF 版本。

## 主要功能

### 1. 批量计算 ED2K

- 支持添加文件或文件夹，批量进入计算队列。
- 支持大批量文件后台入队，减少 UI 卡顿。
- 支持暂停、继续、取消计算。
- 支持复制全部结果或复制选中结果。
- 支持对选中项重新计算、修改 HASH、还原 HASH。

### 2. 历史记录管理

- 自动记录历史计算结果。
- 支持按时间、关键词、大小范围筛选。
- 支持复制链接、复制文件路径、打开原文件所在目录。
- 支持导出选中记录或全部记录。
- 支持直接从历史记录右键添加到资源库。

### 3. 资源管理

- 支持把 ED2K 链接导入到资源库统一管理。
- 支持板块、子集、标签、备注、收藏等整理方式。
- 支持从计算页面和历史记录页面一键添加到资源库。
- 支持资源拖拽移动、批量操作、去重、数据库导入导出。
- 资源表格采用虚拟化方案，适合大批量数据浏览。
- 大量导入时支持后台批量写库，尽量避免前台界面卡死。

### 4. 体验与界面

- 基于 `WPF-UI` 的现代化桌面界面。
- 支持浅色 / 深色主题。
- 支持托盘、任务栏进度、通知提示等桌面交互。
- 多个表格页面使用虚拟表格方案，优先保证性能。

说明:

- 项目当前是 WPF 主线，但为了表格性能，部分高性能表格区域采用了 `WindowsFormsHost + VirtualMode DataGridView` 的混合方案。
- 历史记录和资源管理数据使用 `SQLite` 持久化。

## 适用场景

- 批量计算文件的 ED2K 链接。
- 整理自己长期积累的 ED2K 资源。
- 把“临时计算结果”沉淀为“可归档、可检索、可分类”的资源库。
- 对历史记录进行筛选、导出和二次整理。


## 数据说明

- 历史记录和资源库数据会保存在程序数据目录中。
- 资源管理支持数据库备份、导入和导出，方便迁移。

## 这个项目现在重点解决什么问题

相较于旧版本，这一版更关注下面这些实际问题：

- 大量文件进入队列时的界面响应速度。
- 大量历史记录和资源数据的表格浏览性能。
- 大批量资源导入时的数据库写入卡顿。
- 计算结果、历史记录、资源库三者之间的联动效率。
